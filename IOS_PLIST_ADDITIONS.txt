SINISTER LOCALES — iOS Info.plist additions
============================================

Add these keys to ios/App/App/Info.plist BEFORE the closing </dict> tag.
Without them the App Store build will be rejected and/or location/notifications
will silently fail at runtime.


<!-- Foreground location, shown when user first taps a category or opens app. -->
<key>NSLocationWhenInUseUsageDescription</key>
<string>Sinister Locales uses your location to show you nearby haunted, historical, and macabre sites.</string>

<!-- Background location, shown when user grants foreground and then opens
     the app for a second session, OR when we explicitly request the upgrade.
     iOS REQUIRES both keys (WhenInUse + AlwaysAndWhenInUse) for the
     "Always" permission to be available. -->
<key>NSLocationAlwaysAndWhenInUseUsageDescription</key>
<string>Sinister Locales needs background location access to notify you when you're near a sinister site, even when the app is closed.</string>

<!-- Older iOS versions (<11) — keep for compatibility, App Store may complain
     if missing even though modern iOS uses the AlwaysAndWhenInUse key. -->
<key>NSLocationAlwaysUsageDescription</key>
<string>Sinister Locales needs background location to ping you when you're near a sinister site.</string>

<!-- Required for background location to actually run when the app is not
     in the foreground. Add as an array of background mode strings. If the
     <key>UIBackgroundModes</key> key already exists in Info.plist, MERGE
     these strings into the existing <array>; do not add a duplicate key. -->
<key>UIBackgroundModes</key>
<array>
    <string>location</string>
    <string>fetch</string>
</array>

<!-- Camera-only on-site photo submission. -->
<key>NSCameraUsageDescription</key>
<string>Sinister Locales uses your camera so you can take a verification photo when submitting a new sinister site.</string>


NOTES
-----
- Do NOT add NSPhotoLibraryUsageDescription. Submissions are camera-only by
  design (proof you were on-site). Including the photo library key would
  invite App Store reviewers to ask why we declared it.

- The "Always" location prompt is shown by iOS only after the user has
  used the app once with "While Using" granted. Don't expect it on first
  launch. This is normal Apple behavior.

- After editing Info.plist run `npx cap sync ios` to update the iOS project,
  then rebuild via Codemagic or Xcode.
